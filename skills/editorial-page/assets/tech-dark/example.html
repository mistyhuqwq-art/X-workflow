PRINCIPLES

Three things we won't do

反过来说比较清楚:flux 不做什么。

01

No magic migrations

flux 不会替你 "智能识别" schema 改动。你写什么 SQL,它跑什么 SQL。宁可啰嗦,不要惊喜。

philosophy
02

No hidden state

所有迁移历史用一张普通表记录,字段是人类可读的。不用专有格式、不加密、不压缩。

transparency
03

No "auto-fix"

迁移失败时 flux 会回滚并停下,不会尝试"修复"或跳过。你比它更懂这个库的语义。

safety
"Most migration tools break at the worst possible moment. We tested for that moment first."
— flux team, 内部 RFC-003 · 2025-11

USAGE · 30 SECONDS

看代码比听解释快

本节省略英文官方文档链接,只展示三条最常用命令。

# 新建迁移 (会生成 up.sql / down.sql 两个文件) flux new add_user_email_index # Dry-run: 显示会执行的 SQL,不真跑 flux up --dry-run --env=production # 真跑 + 自动回滚 (任何一步失败则整体 rollback) flux up --env=production --atomic # 查看迁移历史 (纯 SQL 表,可用任何 DB 工具打开) SELECT * FROM flux_history ORDER BY applied_at;

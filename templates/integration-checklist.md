# 联调 Checklist - {功能名}

**日期**:{日期}
**范围**:{涉及的页面/接口}

## Phase 0:准备就绪

- [ ] 字段映射表已产出并确认(链接:...)
- [ ] 环境配置正确(本地 proxy / Staging Cookie)
- [ ] 后端联调节点已确认(对方在线)
- [ ] 已读教训 Top 5(见 frontend-integration skill)

## Phase 1:核心流程端到端

每个功能至少跑通一条完整路径(增/查/改/删):

- [ ] 列表页:加载 → 分页 → 筛选 → 查看
- [ ] 创建页:填表 → 提交 → 跳回列表
- [ ] 编辑页:回显 → 修改 → 保存 → 列表更新
- [ ] 删除:确认 → 删除 → 列表刷新

## Phase 2:字段验证(逐个接口)

对每个接口,打开 Network 面板逐项验证:

### 接口:{接口名}

- [ ] **请求 URL** 正确
- [ ] **Method** 正确(GET / POST / PUT / DELETE)
- [ ] **请求头** 含正确的 Authorization / Cookie
- [ ] **请求 body** 字段名与后端一致
- [ ] **请求 body** 字段类型正确(string/number/boolean/array)
- [ ] **响应结构**与 TypeScript 类型定义匹配
- [ ] **响应字段**用对了(JSON 字符串 parse、枚举值对齐)
- [ ] **时间字段**时区转换正确

## Phase 3:错误处理

- [ ] 400 错误:后端 message 透传到 UI(不是吞成"提交失败")
- [ ] 401 错误:跳转登录
- [ ] 403 错误:提示无权限
- [ ] 500 错误:友好提示 + 不崩溃
- [ ] 超时:有 loading 取消机制
- [ ] 断网:有兜底提示

## Phase 4:边界场景

- [ ] 空数据:列表页空态正常
- [ ] 单条数据:分页不出现问题
- [ ] 大量数据(1000+):性能可接受
- [ ] 特殊字符:名称含 `<>"&'` 不报错
- [ ] 长文本:超长不撑破布局
- [ ] 并发操作:快速点击提交按钮不重复提交

## Phase 5:产出确认

- [ ] 字段映射表已保存到 `knowledge-base/integration/`
- [ ] 新发现的教训已追加到 `knowledge-base/lessons.md`
- [ ] 涉及的 commit 都 push 了
- [ ] `tsc -b && vite build` 通过

The Core Idea

共享上下文是协作的基础

不是给每个人配一个 AI 助手——是让 5 个专业 Agent 和人类在同一个空间里工作， 共享需求、设计、代码、测试的全部上下文。

定义

AI 是团队成员，不是工具栏按钮

每个 Agent 有专业视角和独立判断，参与讨论、产出文档、追踪进度。人类随时可以打断和纠正。

动因

信息碎片化是团队协作的最大成本

需求在飞书、设计在 Figma、代码在 GitLab、测试在 Excel——Teamspace 把它们拉到同一个上下文里。

路径

渐进式集成：链接 → 抓取 → 同步

先放链接聚合信息，再自动抓取内容，最终双向同步。不过度工程，每一步都能独立交付价值。

Capabilities

六大核心能力

从群聊讨论到文档沉淀、代码浏览、进度追踪，覆盖项目全生命周期。

群聊

多角色 AI 讨论

Agent 互相 @mention 展开自动讨论（最多 5 轮），流式输出，人类随时打断。系统智能编排参与者。

知识

四列资源看板

需求 / 设计 / 开发 / 测试四列看板。Markdown 编辑，文件上传，飞书文档一键导入，自动注入 Agent 上下文。

集成

飞书文档读写

Agent 直接读取、追加、创建飞书文档。MCP 协议 + REST API 双通道，文档自动归档到看板。

进度

功能流水线

需求 → 设计 → 开发 → 测试 → 完成，五阶段流转。支持优先级标签、拖拽移动、关联文档。

代码

仓库浏览器

多仓库管理、Git 浅克隆、文件树浏览、提交历史查看。支持 GitLab MR diff 集成。

实时

SSE 事件推送

消息、文档、成员状态全部实时同步。在线指示灯，桌面通知，标题闪烁提醒。

Highlights

几个值得一看的细节

不止是功能堆砌，这些设计决策让 Teamspace 真正好用。

Agent 自主讨论

你发一句，它们聊五轮

PM Agent 分析完需求后 @前端，前端评估完技术方案 @后端确认接口，后端写完又 @测试要测试用例——全自动，最多 5 轮。你觉得跑偏了？发条消息，立刻打断。

飞书 Tool Use

Agent 直接操作飞书文档

不是"帮你生成一段文字让你自己粘贴"。是 Agent 自己打开飞书文档、读取内容、追加结论、甚至新建一份技术方案——然后自动把链接归档到看板。

智能上下文

50 条消息后自动摘要

对话太长 Agent 会变慢变笨？不会。超过 50 条时系统自动把旧消息压缩成结构化摘要，Agent 始终在"摘要 + 最近 20 条"上工作，又快又准。

零门槛接入

局域网打开就能用

没有注册登录，没有复杂配置。同一个 WiFi 下，浏览器输入 IP 地址，起个名字选个角色，3 秒进入项目空间。SQLite 零运维，开箱即用。

Roadmap

下一步，往哪走

MVP 已跑通核心链路，接下来让它变成真正好用的团队工具。

Agent 进化

个人 Agent · 自定义角色

打破 5 个固定角色。每人绑定自己的 Agent——起名、调 prompt、选专业领域。基于模板创建，越用越懂你。四阶段渐进：认领 → 个性化 → 模板商店 → 长期记忆。

基础设施

用户认证 · 飞书 SSO

当前靠 localStorage 存身份，换设备就丢。接入飞书 SSO 后一个账号多端同步，也为权限控制和操作审计打基础。

协作增强

讨论导出 · 通知中心

讨论记录一键导出 Markdown 或飞书文档。站内通知中心替代纯浏览器通知，未读消息不再漏掉。

体验

移动端适配 · 文档协同

响应式布局让手机也能参与讨论。资料库文档从单人覆盖升级为多人协同编辑，不再互相踩。

闭环提效

Demo 浏览器 · 需求即见即评

需求确定后，Frontend Agent 实时构建原型 Demo，直接嵌入右侧预览窗口。在 Demo 旁边圈选元素、留下评论、提出修改意见，Agent 实时响应并迭代。设计评审、交互验收、需求对齐不再靠截图转发——讨论 / 原型 / 反馈在同一屏内闭环。

远期愿景

AI Pet · 你的 Agent 不只是工具，是伙伴

给你的 Agent 一个形象——换皮肤、加表情包、养成个性。用得越久越有默契，看得越久越有感情。它带着你的记忆、偏好和专业判断，在你不在的时候替你参与讨论、review 方案。最终每个人的 Agent，就是自己在数字世界的映射。

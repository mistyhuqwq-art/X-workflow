# Source Registry —— 资料源清单(方法论模板)

> workflow-dlc 依赖外部资料的**管理方法论**。每个团队/个人需要根据自己的资料源,填充本清单的私有版本。

> ⚠️ **隐私提醒**:真实资料源清单(含内部文档链接)**不要放进公开仓库**。按"本清单本地化"一节操作。

## 🎯 为什么要有这个清单

workflow-dlc 的**教训库、模板、方法论**都来自真实实战资料。资料会持续更新。如果不管理,会逐渐过时。

本清单解决 3 个问题:
1. **知道依赖什么**——哪些资料是 workflow-dlc 的源头
2. **知道影响什么**——某份资料更新了,要动哪些 skill
3. **知道何时同步**——上次同步时间 + 下次检查节奏

---

## 📚 资料源分类方法

按**对 workflow-dlc 的影响度**分级:

| 级别 | 含义 | 建议处理 |
|---|---|---|
| **P0 核心源** | 直接决定 skill 结构或方法论,改了就得改 skill | 每月检查 |
| **P1 实战源** | 提供教训和案例,改了要更新 lessons | 每季度检查 |
| **P2 参考源** | 启发性材料,改了可选更新 | 半年看一次 |

---

## 📋 每份资料的登记字段(模板)

```markdown
### {资料名} {图标}

| 字段 | 值 |
|---|---|
| **链接** | {飞书/Confluence/本地路径} |
| **类型** | 飞书 docx / 飞书 wiki / 本地 MD / 外部 URL |
| **首次引用** | YYYY-MM-DD |
| **上次同步** | YYYY-MM-DD |
| **当前版本** | v1.0(文档内部版本号) |
| **字符量** | ~XK(用于快速判断变化) |
| **影响 skill** | 列出所有相关 skill |
| **影响模板** | 列出所有相关模板 |
| **影响教训** | lessons/by-role/xxx.md 里哪几条 |
| **同步节奏** | 每月 / 每季度 / 每半年 |
| **检查重点** | 变更时重点看什么章节 |
```

## 🏗️ 典型资料源类型(参考)

一个成熟的 workflow-dlc 团队通常有以下类型的资料源:

### 类型 1:方法论/规范文档(P0)

- **PRD 规范**:决定 PM skill 的产出结构
- **设计工作流规范**:决定设计师 skill 结构
- **Agent 设计方法论**:决定 Agent 设计师 skill 结构
- **代码规范文档**:决定前后端 skill 结构

### 类型 2:实战复盘(P1)

- **项目完整复盘**:提供教训来源
- **SOP 手册**:提供 skill 的流程依据
- **失败案例库**:提供"不做什么"的依据

### 类型 3:样例/参考(P2)

- **PRD 样例**
- **技术方案样例**
- **设计稿样例**
- **测试用例样例**

## 📅 同步 SOP

### 月度同步(P0 资料,每月 1 次)

**触发**:每月第 1 周。

**动作**:
1. 逐份拉最新版本(用 MCP 或手动)
2. diff 和上次同步版本(通过字符量快速判断大改动)
3. 如有大改动 → 手动 review,更新 skill + 教训
4. 更新本清单的 "上次同步" 和 "当前版本"

### 季度同步(P1 实战源,每季度 1 次)

**触发**:每季度末。

**动作**:拉最新版,重点看"新增教训"章节,把新教训合入 `lessons/by-role/*.md`。

### 半年同步(P2 参考源)

**触发**:半年一次。

**动作**:快速扫一眼标题,有重大变化才深入。

## 🚨 高风险提示

### 本地文件的风险

本地 `~/Downloads/` 等位置的文件:
- 作者更新不会自动感知
- 建议:下次同步时问作者有没有新版,或迁移到共享平台

### 外部依赖的风险

- 外部设计中心维护的 Token 系统可能下线
- 建议:关键值备份到本地知识库

## 📂 本清单本地化(隐私 SOP)

### 为什么本清单只是模板

真实的资料源清单**会包含内部文档链接 / 内部系统名 / 内部人名**,这些**不适合进公开仓库**:
- 飞书文档链接暴露了组织内部信息架构
- 链接本身有时直接通向内部机密
- 即使链接加权限,暴露链接也是信息泄露

### 推荐的本地化方案

```bash
# 1. 在你的私有位置建立本地版(如 ~/Documents/ 或企业知识库)
mkdir -p ~/workflow-dlc-private/sources/

# 2. 复制本模板,改名
cp sources/source-registry.md ~/workflow-dlc-private/sources/source-registry-actual.md

# 3. 在私有版里填真实链接
# source-registry-actual.md 里:
# - 填真实飞书链接
# - 填真实的字符量和版本
# - 填内部流程节奏
```

### 和 workflow-dlc 的配合

- **公开仓库里** 只保留**方法论**(本文件)
- **私有版本里** 保存**具体内容**(source-registry-actual.md)
- 更新 skill 时从私有版读真实内容,公开仓库不需要看到
